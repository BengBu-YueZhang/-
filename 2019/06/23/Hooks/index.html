Hooks（草稿）

Posted on 2019-06-23

什么是Hooks？

Hooks是一种特殊的函数，可以让我们在函数式组件中，使用React的特性。

在Hooks出现之前，我们更喜欢将函数式组件称为无状态组件，出现Hooks后函数式组件，也拥有了state的能力。

StateHook

函数组件中无法使用this，以及this.state。使用useState, 定义了state变量，state变量功能和this.state中的状态功能相同。

useState的参数，就是state变量的初始值。useState的参数可以是数字，字符串，如果你愿意也可使用对象进行初始化。但是记住不像class组件中的this.setState和合并this.state。更新state变量是替换state

如果还需要声明其他变量，只需要再次调用useState即可。

useState还会返回用来更新变量的方法以及state变量。

在函数组件中，我们直接使用变量名，即可以直接使用变量。调用useState返回的更新state变量的方法，即可以更新state。

EffectHook

EffectHook可以在组件中，执行副作用操作。EffectHook作用是替代componentDidMount，componentDidUpdate 和 componentWillUnmount三个生命周期。

什么是副作用，数据请求，设置订阅，修改dom都属于副作用。副作用也分为两种，需要清除的，和无需清除。事件监听就是需要清除的，数据请求是不需要清除的。

useEffect是在渲染后执行的操作，无论是（加载时的渲染（初次渲染），还是更新时的渲染）（当然我们也可以控制他）。react每次运行的时候effect的时候，Dom都已经更新完毕了。

在对dom的事件进行监听时，在class的组件中，需要在componentDidMount生命周期中订阅，并且在componentWillUnmount生命周期中卸载订阅。

使用effect，可以将监听和卸载放在同一个地方。

effect可以返回一个函数，这是effect的清除机制。我们可以将卸载监听的逻辑写到effect返回的清除函数中。

React何时执行清除，在每次执行（渲染）effect时，都会执行清除函数。

Tips

为什么需要EffectHook？

  1. 我们会在生命周期的函数中，添加大量不相关的逻辑。同时又会把相关联的逻辑进行分割。我们可以将不同的副作用，写到不同effecthook中。

为什么每次更新的时候，都需要执行effect

我们可能遇到这种情况，当我们需要根据props的变化，更新组件的state时，我们需要添加componentDidUpdate的生命周期函数。而effect则不需要，当useEffect依赖props时，每次都会自动执行。并执行清除。

effect的性能优化

有的时候，我们可能并不需要执行effect函数，以及清除函数。我们可以为effect指定第二个参数，第二个参数可以是state变量也可以是props。只有前一次渲染，和后一次渲染不同的时候，才会执行effect。

会将第二个参数和之前的进行比较 prev === next 如果是true不执行，如果是false则执行。

传入一个空数组，效果和传入true类似。

Hooks规则

  1. 只在函数组件的最顶层使用hooks.不要在循环，条件或嵌套函数中调用 Hook
  2. 只在 React 函数中调用 Hook, 也可以自定义hook调用其他的hook

说明

在更新的时候，函数会重新执行。

react 会在重复渲染时记住它当前的值，并且提供最新的值给我们的函数。

react是如何知道之前的state，是怎么对应那个useState的。靠的是执行的顺序。所以不能添加循环和判断语句。，

自定义hook

自定义hook可以复用组件的逻辑

自定义hook是一个函数，use开头作为函数名，在自定义hook中可以调用其他hook.
