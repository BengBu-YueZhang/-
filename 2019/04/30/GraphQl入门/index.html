GraphQL入门

GraphQL 是一个用于 API 的查询语言，是一个使用基于类型系统来执行查询的服务端运行时（类型系统由你的数据定义）。

Schema和类型

graphql每一种语言都有自己的实现，graphql文档，将摒除实现上的细节而仅仅专注于其概念。

对象类型和字段

name, appearsIn是对象的字段，String是字符串类型，[Episode]是数组类型，String！表示非空

标量类型

  • Int：有符号 32 位整数。
  • Float：有符号双精度浮点值。
  • String：UTF‐8 字符序列。
  • Boolean：true 或者 false。
  • ID：ID 标量类型表示一个唯一标识符，通常用以重新获取对象或者作为缓存中的键。ID 类型使用和 String 一样的方式序列化；然而将其定义为 ID 意味着并不需要人类可读型。

枚举类型是一种特殊的标量

列表和非空

myField: [String!] 列表可以为空，但是内容不可以为空

myField: [String!]! 列表不可以为空，内容也不可以为空

接口

跟许多类型系统一样，GraphQL 支持接口。一个接口是一个抽象类型，它包含某些字段，而对象类型必须包含这些字段，才能算实现了这个接口。

联合类型

联合类型的成员需要是具体对象类型；你不能使用接口或者其他联合类型来创造一个联合类型。

编写graph的resolvers

什么是resolvers？

resolvers将GraphQl的Query，mutation，subscription转换为数据，要么返回在模式中指定的类型的数据，要么返回该数据的promise。

解析器函数拥有四个参数，parent, args, context, info

  • parent 父类型解析的结果
  • args 传递的参数对象
  • context 所有解析器共享的对象，包含了每一个请求的状态, 身份验证，访问数据源

顶级的解析器第一个参数parent为null, 第二个是查询的参数, 我们可以从第三个上下文对象中获取我们的数据源，方便在解析器中使用。

可以在GraphQL Playground或者postman中进行测试，可以使用下面的查询进行测试，建议为Query或者Mutation添加名称。

对于parseValue，serialize，parseLiteral，含义文档上解释的并不是很清楚。但是stackoverflow上有一个比较好的回答

https://stackoverflow.com/questions/41510880/whats-the-difference-between-parsevalue-and-parseliteral-in-graphqlscalartype

serialize是将值发送给客户端的时候，将会调用该方法

parseValue和parseLiteral则是在接受客户端值，调用的方法

parseLiteral则会对Graphql的参数进行处理，参数会被解析转换为AST抽象语法树。parseLitera会接受ast，返回类型的解析值

parseValue则会对变量进行处理
