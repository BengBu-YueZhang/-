前言

本文正如标题所见，是一篇读书笔记。记录下我所认为在书中比较重要的知识点，并用我的浅薄的知识做了一些补充。

第五章 引用类型

Object类型

在使用字面量方式，定义对象的时候，不会调用Object的构造函数

Array类型

与对象一样，在使用数组字面量表示法时，也不会调用 Array 构造函数

常用的方法 解释
push 向数组末尾添加任意数量项，返回数组长度
pop 移除数组的最后一项，返回被移除
shift 移除数组的第一项，返回被移除项
unshift 向数组前端添加任意数量项，返回数组长度
reverse 反转数组，返回排序后的数组
sort 数组排序，返回排序后的数组
concat 连接多个数组，或者将内容添加到数组的末尾，返回链接后的新数组
slice 根据开始位置和结束位置截取新的数组，返回截取的新数组
splice splice(0, 2)删除从位置0开始的2项。splice(2, 0, 1)在位置2插入1; 返回被删除元素组成的数组。
indexOf 查找数组，如果没有找到返回-1，比较过程中会使用“===”操作符
every 传入的函数必须对每一项都返回true，这个方法才返回true
some 只要传入的函数对数组中的某一项返回true，就会返回true
filter 过滤数组，返回新的数组
map 处理原始数组，返回新的数组
forEach forEach没有返回值
reduce 接收一个函数作为参数，函数接收4个参数，前一个值、当前值、项的索引和数组对象。这个函数返回的任何值都会作为第一个参数自动传给下一项。

函数声明和函数表达式

解析器会率先读取函数声明，并使其在执行任何代码之前可用(可以访问); 至于函数表达式，则必须等到解析器执行到它所在的代码行，才会真正被解释执行。

函数内部属性

内部属性 解释
arguments 函数参数的集合
arguments.callee 指向了拥有arguments对象的函数
arguments.caller 始终为undefind, 主要为了区分函数的caller属性
caller 调用当前函数的函数的引用，如果是在全局作用域中调用当前函数，它的值为null

常见的属性 解释
length 函数的参数的个数
call() 在特定作用域调用函数，第一个参数同apply类似，而call必须将参数全部列出来
apply() 在特定作用域调用函数，接收2个参数，第一个是函数新的作用域。第二个是参数数组，可以是数组或者arguments类数组
bind() 指定特定作用域，返回原函数的拷贝

基础包装类型

基础类型比如字符串，布尔值，数字本身是没有方法的，但是却能调用对应的方法，这是为什么呢？这是因为javascript后台会自动帮我创建基础包装类型。

基础包装类型和引用类型最大的区别是，变量的生命周期。基础包装类型用完即销毁，所以我们无法为基础包装类型添加属性。

基础包装类型 Boolean类型

一般不建议使用Boolean对象, 基础类型Boolean和Boolean对象差异很大。

基础包装类型 Number类型

同样不建议直接使用Number对象，原因同Boolean类型一样

常用的方法 解释
toFixed 返回指定小数位的字符串
toExponential 返回科学计数法的字符串

基础包装类型 String类型

常用的方法 解释
charAt 返回指定位置的字符编码
charCodeAt 返回指定位置的字符串
concat 链接字符串
slice 截取字符串，第一个参数代表开始的位置索引，第二参数代表结束的位置索引
substring 截取字符串，第一个参数代表开始的位置索引，第二参数代表结束的位置索引
substr 截取字符串，第一个参数代表开始的位置索引，第二参数代表截取的长度
indexOf 从头开始查询字符串
lastIndexOf 从末尾开始查询字符串
trim 截取删除首位的空格
toLowerCase 小写
toUpperCase 大写
static fromCharCode 返回编码的字符串

windows对象

全局作用域

全局变量和windows对象上的属性有何区别？

1. 全局变量无法通过delete操作符删除，而windows属性可以
2. 使用未声明的变量会抛出错误，而使用未定义的windows属性则不会

窗口位置

使用window.screenLeft, window.screenTop获取浏览器窗口，距离左上角的位置。

浏览器视口大小

setTimeout和setInterval

setTimeout, setInterval, 可以用来执行javascript字符串, setTimeout(“alert(‘i love you’)”)。

第二个参数是等待的时间，但是经过该时间代码不一定会执行。javascript是单线程解释器，javascript任务队列顺序决定代码执行的顺序，setTimeout以及setInterval的第二个参数是将当前任务push到任务队列中的等待时间。如果队列不是空的，需要等待队列中的代码执行完成后再执行。

setTimeout, setInterval的代码在全局作用域中执行，函数中this的值，在非严格模式下指向window，在严格模式下指向undefined

location对象

属性 解释
hash #号后面的字符串
host 服务器名称和端口号
hostname 服务器名称，不含端口号
href 当前页面的url地址
pathname url中路径的部分
protocol 当前的协议
search 查询字符串”?”开头
port 端口号

除了location.hash属性外，任何location的属性的修改都会触发页面的重新加载。

方法 解释
assign 导航到指定的url，并创建历史记录。效果和window.location.href = url等同
replace 导航到指定的url，但是不会产生新的历史记录
reload 重新加载页面。如果指定参数为true，则从服务器重新加载

如果assign，replace指定的url相对的与当前页面只修改了hash，也不会触发页面的重新加载。

修改hash值会触发onhashchange事件，但是通过pushState，replaceState修改不会触发事件

第十章 DOM

HTML中每一个标记都可以通过一个节点表示，共有12种类型节点，这些类型都继承于一个基类。

Node类型

javascript的所有节点类型，都继承自Node类型。以下12个常量表示12种类型。

  • Node.ELEMENT_NODE(1);
  • Node.ATTRIBUTE_NODE(2);
  • Node.TEXT_NODE(3);
  • Node.CDATA_SECTION_NODE(4);
  • Node.ENTITY_REFERENCE_NODE(5);
  • Node.ENTITY_NODE(6);
  • Node.PROCESSING_INSTRUCTION_NODE(7);
  • Node.COMMENT_NODE(8);
  • Node.DOCUMENT_NODE(9);
  • Node.DOCUMENT_TYPE_NODE(10);
  • Node.DOCUMENT_FRAGMENT_NODE(11);
  • Node.NOTATION_NODE(12);
节点属性 解释
nodeType 节点的类型，值 1 ～ 12
nodeName 不同类型的节点，有所不同
nodeValue 不同类型的节点，有所不同
childNodes 保存NodeList对象，是一个类数组。包含节点的直接子元素
parentNode 节点文档树中的父节点
previousSibling 前一个同胞节点
nextSibling 下一个同胞节点
firstChild 第一个子节点
lastChild 最后一个子节点
hasChildNodes() 判断是否拥有子节点
ownerDocument 指向文档节点（html元素）

常见的方法 解释
appendChild 向childNodes列表的末尾添加节点，如果是已有的节点则会改变它在文档中的位置
insertBefore(newNode, 指定元素) 插入指定元素的前面，如果指定元素为null。insertBefore的行为同appendChild一致。如果是已有的元素，则会移动已有的元素。
replaceChild(newNode, 被替换的节点) 替换节点，被替换的节点的关系指针都会被复制到新的元素上
removeChild 删除指定节点
cloneNode 克隆节点，如果指定参数为true，则会深拷贝。不会拷贝javascript属性，如事件处理程序。

Document类型

document是HTMLDocument（继承Document类型）的实例。nodeType类型为9。

常见的属性和方法 解释
documentElement documentElement属性指向html元素
body 指向body元素
doctype <!DOCTYPE>的引用
title 浏览器标签
URL 当前页面的url地址
domain 当前页面的域名
images 页面中所有img元素
createElement 创建元素节点
createTextNode 创建文本节点

类似getElementsByTagName的方法会返回一组节点的引用，会返回HTMLCollection对象。HTMLCollection对象同NodeList类似，也是一个类数组。

Element类型

  • nodeType等于1
  • nodeValue始终为null
  • nodeName，tagName为标签名
常用的属性和方法 解释
getAttribute(key) 获取特性。而style和事件使用getAttribute将会获得字符串。直接访问属性，会获得对象和函数。自定义特性，需要添加data-前缀。
setAttribute(key, value) 设置特性，可以是元素的特性，也可以是自定义特性。
removeAttribute(key) 删除特性
attributes 返回一个NamedNodeMap, 包含了节点的所有特性，是一个类数组。nodeName是NamedNodeMap节点中的key，nodeValue是NamedNodeMap节点中的value。

Text类型

  • nodeType等于3
  • nodeName等于”#text”
  • nodeValue等于文本内容
  • 没有子节点
  • 可以使用在父节点上调用normalize()可以合并多个文本节点

Comment类型（注释类型）

  • nodeType等于8
  • nodeName等于”#comment”
  • nodeValue等于注释内容

关于NodeList

关于NodeList以及NamedNodeMap, HTMLCollection.这三个集合都是动态的。每当文档产生变化，它们都会更新。应当减少对NodeList的使用，每一次引用NodeList，都会运行一次基于文档的查询。
