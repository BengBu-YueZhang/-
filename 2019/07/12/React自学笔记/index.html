「又双叒叕」React自学笔记

Posted on 2019-07-12

前言

为什么说「又双叒叕」这是因为这是我第三次完整阅读react的文档了，之前虽然有学习过，但是在工作中长时间没有使用所以很多概念都有所遗忘，并且Context，Ref等API从16.3版本多次改动，所以准备花几天的时间重新阅读整理下React的知识点。

⏰因为这里主要都是文档中的内容，主要自个做个记录。在掘金专栏中我只会放在阅读栏目里做一个备份。并不希望其他人看到

主要概念

JSX

JSX的本质是React.createElement的语法糖🍬。推荐使用小括号包裹JSX，避免分号自动插入的Bug。在JSX中使用{}插入JavaScript表达式以及属性。

下面是JSX的模拟编译解析结果👇。由此可知在jsx文件中React虽然没有被直接引用，但是在文件开头必须声明React变量。

编译为

当我们没有给JSX中的属性传值时，属性默认为true。JSX的属性可以使用扩展运算符, 传递对象的全部属性。

使用props.children属性获取JSX中的子代。布尔值，null和undefined在JSX中会被忽略。

元素渲染

元素是构成React最小单位。可以使用ReactDOM.render将元素渲染到DOM节点中。通过Diff算法，React只会更新必要的部分。

组件 & Props

React中可以使用函数或者class定义组件

组件的Props是只读的。为什么只读的，可以参考「React」函数组件与Class组件有何本质不同？

State

使用this.state为组件创建局部状态。使用useState返回的方法或者this.setState修改状态。

  1. 不要直接修改状态
  2. 状态更新可能是异步的
  3. this.setState是合并更新
  4. 数据自上而下流动

事件处理

  1. 必须使用preventDefault阻止默认行为
  2. 如果类的方法是JSX的回调函数，默认是不会绑定this的。this为undefined。
