Node多进程模型

参考5分钟node多进程模型从入门到精通, 系列３｜走进Node.js之多进程模型, Multi threading and multiple process in Node.js

什么是进程？

进程是系统进行资源分配和调度的基本单位。进程是线程的容器。进程拥有独立的内存空间，一个进程无法直接访问另一个进程。

什么是线程？

线程是CPU调度和分配的基本单位，是比进程更小的单位。线程不具有独立的系统资源，可以与同属同一个进程的线程共享一部分资源。但是堆栈、局部变量等资源是独有的。同一个进程中可以有多个线程可以并发执行。

进程和线程的关联？

一个程序至少包含一个进程，一个进程中至少包含一个线程。

单线程（node）和多线程（java）是如何处理并发的？

  • java使用多个线程同时处理多个请求，但是大部分时间cpu的占用率为0%，在等待数据库的返回
  • node使用事件循环和异步io处理并发。但是如果存在cpu密集操作，会阻塞事件循环。

node的”单线程”

node的单线程指的是js执行是单线程。Node启动后会创建V8实例，而V8实例是多线程。

node的子进程

子进程的好处

  1. 将耗时的任务交给子进程处理，可以避免阻塞事件循环。
  2. 可以在子进程中执行shell命令。

node可以通过cluster.fork开启多进程，多进程可以共享tcp链接，自带负载均衡。node中cluster的源码也是基于child_process.fork封装的。

Node process模块

参考Mastering the Node.js Core Modules - The Process Module, Node.js v10.16.0 文档

process是一个全局对象，提供了获取当前Node主进程的信息，可以直接使用，无需require。process对象是EventEmitter的实例。所以可以使用.on订阅事件。

进程事件

beforeExit

beforeExit会在Node清空事件循环时触发, 但是可以在beforeExit的callback的内部可以注册事件监听，进行异步调用。从而导致Node进程继续。

exit

Node事件循环不在执行任何操作，或者主动调用process.exit()方法时，触发。

message

当子进程收到父进程使用childprocess.send发送的消息时触发。

multipleResolves

multipleResolves事件会在，Promise中resolve多次，reject多次，resolve后进行reject，reject后进行resolve时触发。

rejectionHandled

unhandledRejection

当Promise被reject，但是Promise没有绑定catch错误处理器，unhandledRejection事件会被触发。

uncaughtException

当异常未被捕获的时候，会触发uncaughtException事件。

  1. uncaughtException中的错误无法被捕获
  2. 不要尝试在uncaughtException中尝试恢复应用.

warning

warning事件，会在Node发出进程警告时触发
