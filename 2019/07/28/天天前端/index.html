2019.7.18 执行上下文栈

参考冴羽的博客; 汤姆大叔的博客

什么是可执行代码（executable code）？

ECMAScript中有三种可执行代码(global, function, eval)；可执行代码的执行依赖执行上下文；活跃的执行上下文在逻辑上会形成一个堆栈（执行上下文堆栈），执行上下文堆栈的底部永远是全局上下文。执行上下文堆栈的顶部永远是当前活跃的上下文。

什么是执行上下文（Execution Context）？

执行上下文可以抽象理解为一个对象。上下文有一系列属性(状态)。

  1. variable object，变量对象
  2. this value，this指针
  3. scope chain，作用域链

变量对象（variable object）

定义，变量对象(variable object) 是与执行上下文相关的 数据作用域(scope of data) 。它是与上下文关联的特殊对象，用于存储被定义在上下文中的 变量(variables) 和 函数声明(function declarations) (不包含函数表达式)。

在不同的上下文中，变量对象表示不同的object。但是在全局上下文中，变量对象也是全局对象自身。所以全局上下文中的变量也是全局对象的属性。

什么是执行上下文栈（Execution Context Stack）？

一个执行上下文可以激活另一个上下文。例如：函数A内部调用了函数B，函数A的上下文激活了函数B的上下文。函数B也可以激活函数C的上下文。类似数据结构中的栈。激活其他上下文的上下文称为调用者（caller），被激活的上下文被称为被调用者（callee）。callee同时也能是caller。当caller激活callee时，caller会暂停执行，控制权将会交给callee。当callee执行完成后，caller会再次执行。

当程序启动，全局上下文就会推入执行执行上下文堆栈。

2019.7.20 Node多进程模型

参考5分钟node多进程模型从入门到精通, 系列３｜走进Node.js之多进程模型, Multi threading and multiple process in Node.js

什么是进程？

进程是系统进行资源分配和调度的基本单位。进程是线程的容器。进程拥有独立的内存空间，一个进程无法直接访问另一个进程。

什么是线程？

线程是CPU调度和分配的基本单位，是比进程更小的单位。线程不具有独立的系统资源，可以与同属同一个进程的线程共享一部分资源。但是堆栈、局部变量等资源是独有的。同一个进程中可以有多个线程可以并发执行。

进程和线程的关联？

一个程序至少包含一个进程，一个进程中至少包含一个线程。

单线程（node）和多线程（java）是如何处理并发的？

  • java使用多个线程同时处理多个请求，但是大部分时间cpu的占用率为0%，在等待数据库的返回
  • node使用事件循环和异步io处理并发。但是如果存在cpu密集操作，会阻塞事件循环。

node的”单线程”

node的单线程指的是js执行是单线程。Node启动后会创建V8实例，而V8实例是多线程。

2019.7.27 作用域链（Scope Chain）

参考汤姆大叔的博客

作用域链是执行上下文中的另一个重要属性，作用域链是执行上下文所有变量对象（variable object）的列表。

上面的代码中，包含3个执行上下文（全局上下文，foo函数的上下文，bar函数的上下文）。bar的执行上下文的作用域链中包含了3个变量对象（variable object）。bar活动对象，foo活动对象，全局变量对象。

作用域链是在函数调用时创建的, 包含了函数的活动对象（AO），以及函数内部的[[scope]]属性。scope的定义：作用域链（scope）= AO + [[scope]]属性。

函数可以访问，更高层次的执行上下的变量对象，这种机制正是通过[[scope]]属性实现的。[[scope]]属性是所有父变量对象的层级链。

作用域的变化过程

函数创建阶段

JavaScript是词法作用域（静态作用域）的语言。[[scope]]属性在函数创建时就已经写入并存储在函数对象中。即使函数不会被调用，[[scope]]属性也会被创建。

函数激活阶段

当函数被激活，创建执行上下文的AO属性后，上下文的作用域链等于：作用域链（scope）= AO + [[scope]]属性。活动对象（AO）是作用域数组的第一个对象。这个特性对于标识符解析很重要，因为变量名的查找中，是从最深的上下文（自下而上）查找的。局部的变量，相较于父作用域的变量，拥有较高的优先级。

2019.7.28 this

参考汤姆大叔的博客, 冴羽的博客

this是执行上下文中一个重要属性，this在进入上下文时确定，并且在上下文运行期间永久不变。

全局代码中的this

在全局代码中this就是全局对象本身。

函数代码中的this

函数代码的this不是静态绑定的，而是在进入上下文时才会确定。但是在函数代码在运行时，this是不变的。

Reference

ECMAScript的类型分为语言类型和规范类型。语言类型指的是常用的Undefined、Null、Boolean、String、
Number、Object。规范类型指的是底层用来描述语言类型的包含，Reference、List、Completion、Property Descriptor、 Property Identifier、Lexical Environment、Environment Record。

Reference是只存在于规范中的抽象类型，为了更好地描述语言的底层行为逻辑。Reference由三部分构成

  • base value，拥有属性的那个对象，值可能是Undefined，Object，Boolean，String，Number，EnvironmentRecord
  • referenced name，属性名
  • strict reference 是否为严格引用（非严格模式下，如果this为undefined会被隐式转换为window）

举个例子

规范中提供了获取Reference组成的方法

  • GetBase 获取Reference的base
  • IsPropertyReference 如果Reference的base是一个对象返回true
  • GetValue 从Reference类型获取对应值的方法

如何确定this的值？

  1. 计算MemberExpression的结果赋值给ref
  2. 判断ref是不是一个Reference类型
    1. 如果ref是Reference，并且IsPropertyReference(ref)是true, 那么this的值为GetBase(ref)
    2. 如果ref是Reference，并且base值是EnvironmentRecord, 那么this的值为ImplicitThisValue(ref)
    3. 如果ref不是Reference，那么this的值为undefined

什么是MemberExpression？

MemberExpression可以简单理解为()左边的内容
