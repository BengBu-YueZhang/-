前言

本文正如标题所见，这是一篇读书笔记。记录下我所认为在书中比较重要的知识点，并用我的浅薄的知识做了一定的补充。欢迎大家指正。

阅读高程这本书，前前后后花了不到3个星期的时间。书中的一些内容确实有些过时，尤其是讲解了大量对于IE8的处理。但是不能否认的是，高程这本书的经典，和尼古拉斯·泽卡斯的伟大。

第五章 引用类型

Object类型

在使用字面量方式，定义对象的时候，不会调用Object的构造函数

Array类型

与对象一样，在使用数组字面量表示法时，也不会调用 Array 构造函数

常用的方法 解释
push 向数组末尾添加任意数量项，返回数组长度
pop 移除数组的最后一项，返回被移除
shift 移除数组的第一项，返回被移除项
unshift 向数组前端添加任意数量项，返回数组长度
reverse 反转数组，返回排序后的数组
sort 数组排序，返回排序后的数组
concat 连接多个数组，或者将内容添加到数组的末尾，返回链接后的新数组
slice 根据开始位置和结束位置截取新的数组，返回截取的新数组
splice splice(0, 2)删除从位置0开始的2项。splice(2, 0, 1)在位置2插入1; 返回被删除元素组成的数组。
indexOf 查找数组，如果没有找到返回-1，比较过程中会使用“===”操作符
every 传入的函数必须对每一项都返回true，这个方法才返回true
some 只要传入的函数对数组中的某一项返回true，就会返回true
filter 过滤数组，返回新的数组
map 处理原始数组，返回新的数组
forEach forEach没有返回值
reduce 接收一个函数作为参数，函数接收4个参数，前一个值、当前值、项的索引和数组对象。这个函数返回的任何值都会作为第一个参数自动传给下一项。

基础包装类型

基础类型比如字符串，布尔值，数字本身是没有方法的，但是却能调用对应的方法，这是为什么呢？这是因为javascript后台会自动帮我创建基础包装类型。

基础包装类型和引用类型最大的区别是，变量的生命周期。基础包装类型用完即销毁，所以我们无法为基础包装类型添加属性。

基础包装类型 Boolean类型

一般不建议使用Boolean对象, 基础类型Boolean和Boolean对象差异很大。

基础包装类型 Number类型

同样不建议直接使用Number对象，原因同Boolean类型一样

setTimeout和setInterval

setTimeout, setInterval, 可以用来执行javascript字符串, setTimeout(“alert(‘i love you’)”)。

第二个参数是等待的时间，但是经过该时间代码不一定会执行。javascript是单线程解释器，javascript任务队列顺序决定代码执行的顺序，setTimeout以及setInterval的第二个参数是将当前任务push到任务队列中的等待时间。如果队列不是空的，需要等待队列中的代码执行完成后再执行。

setTimeout, setInterval的代码在全局作用域中执行，函数中this的值，在非严格模式下指向window，在严格模式下指向undefined

关于NodeList

关于NodeList以及NamedNodeMap, HTMLCollection.这三个集合都是动态的。每当文档产生变化，它们都会更新。应当减少对NodeList的使用，每一次引用NodeList，都会运行一次基于文档的查询。

第十三章 事件

事件流

事件冒泡

事件开始时由最具体的元素接收，然后逐级向上传播。事件最终会传播到window对象。

事件捕获

事件捕获的是不太具体的节点应该更早接收到事件，然后逐级先下传递。事件最早从window对象向下传递

DOM事件流

DOM事件流共有3个阶段，捕获阶段，处于目标阶段（看作冒泡阶段的一部分），冒泡阶段

事件处理程序

事件处理程序可以通过3种形式添加。

如果使用addEventListener添加事件处理程序，只能通过removeEventListener移除。这意味着，我们无法移除通过addEventListener添加的匿名事件处理程序。并且addEventListener可以为一个事件，添加多个事件处理程序。

addEventListener的第三个参数，可以指定事件处理程序添加到冒泡阶段还是捕获阶段。false将事件处理程序添加到冒泡阶段。true为捕获阶段。

事件对象

在DOM0级或者DOM2级的事件处理程序中，浏览器都会传入event对象到事件处理程序中。在HTML事件处理程序中，变量event保存着事件对象。

事件对象常用的属性和方法 解释
type 事件的类型
currentTarget 指向事件处理程序绑定的元素
target 指向事件触发的元素
eventPhase 调用事件处理程序时，事件所处的阶段。1处于捕获阶段，2处于目标阶段，3处于冒泡阶段
cancelable 是否可以取消默认行为
preventDefault 取消默认行为
bubbles 事件是否会冒泡
stopPropagation 阻止冒泡或捕获
stopImmediatePropagation 阻止冒泡或捕获，同时阻止任何事件处理程序被调用(DOM3级事件新增)
trusted 是否由浏览器生成的事件(DOM3级事件新增)

currentTarget始终等于事件处理程序中的this, 而taget则指向的是事件触发的对象

复合事件

DOM3级事件中新增的一类事件，针对输入法编辑器（IME）。例如在美式键盘上输入中文的时候，我们需要按下多个键，但是最终只输入一个字符。复合事件就是针对这种情况而设计的。

针对复合事件，事件对象新增了一个属性，event.data

  • compositionstart，IME准备输入时触发，event.data是选中的字符。
  • compositionupdate，向IME输入字符时触发, event.data是IME中正在输入的字符。
  • compositionend，IME关闭时触发，event.data是IME插入的所有字符。
