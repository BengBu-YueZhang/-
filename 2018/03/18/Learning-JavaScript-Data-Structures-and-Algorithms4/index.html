链表

为什么需要链表？

数组是存储一组数据最简单的方式，但是在大多数语言中数组的长度是固定的。并且操作数组的性能成本会很高, 虽然在JS中提供了splice这样的方法，但是在背后原理是一致的。修改，删除，添加数组会造成数组的每一项发生移动。会造成不必要的性能浪费。

链表的数据结构

链表的数据结构是一组数据有序的结合，但是在内存中链表中的数据不是连续的。链表的存储方式如下图（链表中每个元素，由一个存储元素本身的节点，和指向下一个元素的引用）

链表与数组的比较

  1. 访问链表中一个元素，需要从起点开始迭代列表直到找到所需的元素，而数组可以直接获取。
  2. 链表添加移除元素的时候不需要移动其他元素的位置

append方法, 向链表中添加元素

append方法分为两种情况：

  1. 如果链表的头部为null，那么创建的元素就是头部
  2. 入股链表的头部不为null. 那么创建的元素就是链表最后一个元素的next所指的对象
