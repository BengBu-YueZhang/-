目前存在的问题

Stack不能创建私有属性, 我们希望Stack类，只能使用我们提供的方法。比如在Stack类，对于Stack的实例，如果我们获取属性items, 我们就可以直接使用数组的方法，操作Stack实例。如果items是一个私有的属性，则不会存在这个问题

私有属性的解决办法: Symbol

Symbol是唯一生成的，所以在实例的外部，无法获取同样的Symbol值，所以可以看作私有属性,但是Symbol不是真正的私有属性，Object.getOwnPropertySymbols可以获取Symbol的key值

私有属性的解决办法: WeakMap

通过闭包，将items包裹在闭包空间内，避免外部对其修改，但是子类无法继承这样的私有属性

汉诺塔问题

💡我的想法：虽然最简单的是三个盘子的汉诺塔，但是面对三个盘子整个递归的过程并不是很好理解脑补出来，可以把问题简化成二个盘子的问题，就会非常容易理解汉诺塔递归的整个过程。(文字不太好描述，需要自己想象一下)

⚠️⚠️⚠️⚠️⚠️特别注意分析问题的时候不要让自己按着程序运行的顺序钻进去，这样会很容易困惑和乱掉
