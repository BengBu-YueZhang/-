TypeScript(1)

发布于 2018-07-25

前言

由于公司的内部项目,是由技术主导开发的而非产品。本着Geek的精神, 想在日志分析系统中使用Vue配合TypeScript作为前端的技术栈。可能因为个人能力有限, TS的文档有一部分内容，个人认为非常晦涩绕口, 我个人在看完前半部分的TS的文档后, 用自己的语言写下这篇博客帮助自己更好的理解TS。

基础类型

枚举

数字枚举以及字符串枚举。数字枚举在没有指定初始化的情况下，从0开始，依次加1递增。异构枚举则是，枚举可以混合字符串和数字成员。但是一般不会这样做。枚举可以指定表达式，会在后面介绍到。

Any

Any指定的是通用的类型, 在非万不得已的情况下, 不要使用Any类型。

Void

表示没有类型, 函数中使用void表示没有返回值

Null Undefined

Null Undefined可以赋值给任何类型的变量

Never

表示异常的返回值, 返回never的函数，拥有不会到达终点

类型断言

我们通常比ts类型检查器更清楚，某一个变量的详细信息, 我们可以通过类型断言指定变量的类型。

interface

接口的作用是为值, 参数, 类, 函数等进行类型检查。

可选属性 ?

interface中定义的属性, 并不完全都是必须的。可以使用?标记为可选属性。定义可选属性后, 可以不传递使用该属性, 但是不能使用interface中没有定义的属性

只读属性 readonly

只读数组 ReadonlyArray

函数类型

interface可以用来描述函数类型, 需要指定参数的类型和返回值的类型

可索引类型

对于一些未知key值的属性，我们该如何在接口中定义它呢?答案是可以使用字符串索引类型。可索引分为字符串索引和数字索引。但是数字索引是等同于字符串索引的，因为对于key值, 100将会转换为”100”

类类型

interface可以强制一个类实现某一种规范。但是interface只定义类的实例的部分，没有定义类的静态的部分

接口继承

接口与接口之间可以相互继承, 可以达到复用接口的目的

接口继承类

interface可以继承class, interface会继承类的成员的定义(实例的属性以及实例的方法), 但是不包含实现。接口可以继承private和protected的成员，如果接口继承了private和protected的成员，那么这个接口只能被这个类以及它的子类所实现。因为private和protected的成员是私有的，private只能在定义的类的内部访问，protected可以在子类中访问。

class

部分与es6内容重合的部分不做介绍

修饰符

public

ts默认包含了public的修饰符, public修饰符可以自由的访问类的属性以及方法

private

当类的成员被标记为private, 被标记为private的成员将不能在类的外部访问，包括了派生类。

⚠️ 🏁 TS是基于结构进行类型兼容的, 但是如果成员包含了private或者protected的修饰符, 即使两个类的结构一样，但是protected或者private的成员不来自同一个类型, 那么两个成员之间也是不兼容的（具体请看类型兼容这一章节）

protected

与private类似，但是区别在于protected的成员，虽然在类的外部无法访问，但是可以在派生类中访问父类的protected的成员。构造函数也可以被标记成protected。这意味着这个类不能在包含它的类外被实例化，但是能被继承。可以在派生类中实例化(super)

readonly

可以设置类的属性，为只读属性

构造函数中使用修饰符参数

在构造函数中使用修饰符参数, 相当于把类的声明与赋值放在了同一个地方

getter, setter

同es6一样

静态属性

在es6中，并不支持静态方法, 但是在typescript中实现类的静态方法。通过[类名].的方式，访问静态属性和静态方法

抽象类 abstract

抽象类，本身不会实例化，派生类通过继承抽象类进行实例化。抽象类与interface最大的区别在与抽象类中可以包括具体的实现细节，而interface只包含了成员的定义。（抽象类的抽象方法不包含方法的具体实现）

类的类型

定义一个Foo类型, Foo类的实例的类型是Foo, 那么Foo本身的类型是什么呢? 答案是可以用”typeof Foo”获取类的类型

function

定义函数类型

可选参数，默认参数

可选参数使用, “?”标记为可选参数, 可选择参数的后面必须没有必选参数, 可选参数必须放在所有参数的最后面。默认参数可以直接指定”=”设置为默认参数，带默认值的可选参数，可以不放在参数list的最后面，在使用的时候传入undefined就可以使用默认参数的默认值。

剩余参数

同ES6一样，未知长度的参数可以使用rest参数。

this

箭头函数与this

箭头函数的this的值，是创建时this的值

this参数

this参数，是假参数，位于参数列表的最前面。用来指定函数内部this的类型。因为在字面量对象，函数中的this类型为any, 而非字面量对象

this参数在回调函数中

如果我们既想函数类型为void，同时this又指向实例呢？可以在类的方法声明的时候使用箭头函数，但是只有在ts中支持类中使用箭头函数

重载

有的时候需要为函数传入各种不同类型的参数，以及不同类型的返回值, 根据不同的参数和返回值需要定义不同的参数声明

泛型

为什么需要使用泛型？

代码需要考虑到可重用性，可以使用泛型，支持多种数据类型

假设遇到以下这种状况, 我们通过的传入和传出值要是类型统一的。如果使用不同类型重载显然是不现实的。如果使用的any, 则会丢失信息。(返回值any, 传入值any，意味着传入是number, 传出是string也是可以的)此时我们可以使用类型变量

泛型变量

类型变量T, 可以是任何类型, 书写泛型函数的时候，请务必考虑到这一点 💭

泛型接口

泛型类

与interface类似, 同样可以指定类型变量参数。可以确认指定的属性使用同样类型的参数

泛型约束

使用interface创建泛型的约束条件

泛型与类类型

使用泛型的类型变量T, 指定类的构造函数的类型(构造函数的返回值)

enum

枚举定义一组带有名字的常量, 如果第一个成员没有被初始化，那么第一个会被赋予0，后面的枚举值会依次递增。如果之前的枚举成员是一个 数字常量, 后面是根据第一个枚举成员依次递增加一的。字符串枚举则是把一些变量，用更加语义化的方法表达。枚举的值同样可以是表达式，如果表达式的计算结果，是NaN，或者Infinity编译后会报错。

反向映射

从枚举的value，获取enum的key。但是反向映射只针对数字枚举

常量枚举

常量枚举使用const修饰符，常量枚举不允许有计算成员，因为在ts完成编译后，枚举会被删除，枚举的值会被嵌入到代码里

类型推论, 类型兼容

💡 推论为联合类型最重要的一点就是类型需要是兼容的

类型兼容

💡 typescript的类型兼容性是基于结构，而非名义的, TypeScript结构化类型系统的基本规则是，如果x要兼容y，那么y至少具有与x相同的属性。但是如果类型有private和protected的属性，只有它和它的子类才是兼容的

函数的兼容性

函数兼容性，比较是参数以及参数的类型，和函数的返回值

可选参数以及剩余参数

可选参数，不会影响到函数的兼容性。…参数也不会影响到函数的兼容性, …参数会被当作无限个可选参数

类

两个类进行兼容性比较的时候，只会对实例部分进行对比。如果类包含private和protected的成员，需要兼容则必须私有成员来着同一个类(继承)

泛型

高级类型

交叉类型

交叉类型是将多个类型合并为一个类型

联合类型

联合类型表示一个值，可以拥有多种类型，使用“ | ”的操作符

如果一个值是联合类型，我们只能访问，两种类型的交集部分

类型保护和区分类型

为什么需要使用类型保护？

如果不使用类型保护，面对联合类型，必须使用类型断言处理联合类型，请看下面的例子

自定义类型保护 ( it is Class )

定义一个函数判断类型，函数的返回值安装规定定义为”it is Class”的形式

typeof保护

typeof保护和自定义类型保护一样，可以避免在每一行的代码中使用类型断言，但是typeof保护只能用来，number，boolean，string类型

instanceof保护

null, undefined

null和undefined可以赋值给任意的类型的参数。同时null与undefined不是一回事，需要用联合类型区分

null, undefined的类型断言

使用 “!” 对null,undefined进行类型断言

类型别名

类型别名的作用与interface类似，对变量和属性的约束。类型别名不能继承不能作为类的接口。尽量使用接口而非类型别名

字符串字面量类型, 数字字面量类型

定义一组字符串, 数字,使用的时候只能从字符串组，数字组中固定的值
