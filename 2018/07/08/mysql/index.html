检索

SELECT

DISTINCT

检索不重复的行, DISTINCT只能用于单独的列, 如果指定多个列将会检索出全部的数据

LIMIT

限制mysql返回的结果集的长度, 基本语法如, LIMIT Number1 OFFSET Number2, Number1指的是返回的结果集的长度, Number2指的是从表的那个位置开始检索

完全限定表名

用来区分不同的表, 字段名却相同的字段。在后面的多表联合查询可以使用到

排序

根据SELECT检索出的数据的顺序是不可靠的，如果不指定排序规则，检索出的数据的顺序是没有意义的

什么是子句

SQL语句是由子句构成的，子句可以是必须的，也可以是非必须的，SELECT, FROM就是属于子句。ORDER BY则属于排序的子句

ORDER BY

对于字符串的排序，mysql采用的是字典序。可以使用ASC(升序), DESC(降序), 指定排序的顺序

  • BETWEEN 之间 (BETWEEN [star] AND [end])
  • IS NULL 为空值

AND

OR

主要事项

⚠️ 🚗 🚌 🚀 🔥 🦁️ 🐶 🐯 🦅 🐱 🐼 🐻

  1. GROUP BY 可以进行嵌套分组，但是数据将会在最细的分组上展示
  2. NULL会被当作单独的分组
  3. 除聚集计算语句外，SELECT语句中的每个列都必须在GROUP BY子句中给出
  4. GROUP BY可以根据表达式进行分组，比如可以是Date()，Year()

WHERE是对行的过滤，HAVING是对聚合结果本身的过滤。类似与elasticsearch中过滤桶

分组排序

不能依赖GROUP BY进行排序数据，应该指定ORDER BY排序数据。因为我们不一定要按照GROUP BY的顺序进行排序。可能会依赖其他顺序排序。

子查询

子查询总是从内向外查询的，由于需要考虑到性能的原因，不建议嵌套过多的子查询。编写子查询很具有技巧性，先编写外层查询，然后用硬编码的形式进行测试，正常后潜入子查询。然后再次测试。逐渐的嵌套，可以减少未来排错的时间。

联结

把信息分解成不同的表，一类数据一个表，各表通过某些常用的值(即关系设计中的关系(relational))互相关联。

A表存储了供应商的相关信息，每一个供应商将拥有一个唯一标示（主键），B表存储了产品的信息，每一条产品信息存储了供应商的主键。除了供应商的主键外，B表不存储任何供应商的相关信息，A表的主键成为称为B表的外键，将A表和B表相关联。外键包含了另一个表的主键值。

创建联结

笛卡尔积

如果在INNER JOIN没有指定ON的条件，两张表联结的时候，将会出现笛卡尔积，既返回的内容等于第一张表*第二张表，第一个表一一对应第二张表的每一行。

内部联结

目前使用的联结又被称为等值联结，基于两个表之间的相等测试。

多表联结

🐒 MYSQL对一个SELECT可以联结的表，没有限制，同两张表一样，联结规则就是列出所有的表，然后定义表和表之间的关系。MySQL在运行时关联指定的每个表以处理联结。这种处理可能是非常耗费资源的，联结的表越多，性能下降越厉害。

自联结

对同一张表进行多表查询的操作。为什么会需要这样做呢?举一个🌰。我们知道一位员工的emp_no等于10001, 我想知道和他同一天出生的员工有谁。可能会遇到这种需求，这个时候我们就可以使用自联结。当然也可以使用子查询的方式

左外联结, 右外联结 OUTER JOIN

查找没有关联的那些行，分为 LEFT OUTER JOIN 和 RIGHT OUTER JOIN（左关联，右关联）, 使用 LEFT OUTER JOIN 就是指定 OUTER JOIN 左边的表的所有的行，RIGHT OUTER JOIN OUTER JOIN 右边的所有的行。左外联结, 右外联结没有特殊的意义，通过交换表名的位置，同样可以做到切换不同的外联。 🇫🇷 🇬🇧

聚合的联结

吐槽：mysql的聚合操作，比elasticsearch不知道慢到哪里去了 😢

组合查询

这些多条SQL语句，将结果作为单个结果集返回。UNION将自动去除多次查询重复的行，而 UNION ALL将会返回相同的行

组合查询的使用场景:

  1. 实际工作中，数据可能会存放在不同的表当中，所以可能需要多张表同时查询
  2. 多单个表进行不同WHERE子句的查询，结果集需要统一返回

UNION需要注意的点:

  1. 必须由两条及以上的SELECT语句
  2. UNION的每一条SELECT语句必须包含相同的列，表达式，聚集函数
  3. 多个查询的列的数据类型必须兼容
  4. 对返回结果只能使用一条，ORDER BY的字句

数据插入

提高整体性能

数据库通常被多个客户访问，INSERT可能会变得非常耗时，会降低SELECT语句的性能, 可以在INSERT INTO之间插入LOW_PRIORITY，提高mysql的整体的性能。同样适用与update，和remove语句，LOW_PRIORITY会降低INSERT INTO的执行权重

插入多个

数据更新和删除

  1. 不要使用不带WHERE语句的更新, 删除语句
  2. 保证每一个表拥有主键（因为指定其他条件，可能会删除多个的情况）
  3. 在UPDATE，DELETE使用前，请用SELECT确认，两者的WHERE子句是否是查询正确的值

更新 UPDATE

如果不指定WHERE字句，UPDATE将会更新全部的行

删除 DELETE

如果不指定WHERE字句，UPDATE将会删除全部的行

  3. TEXT, 变长类型
