前言

redis shell

redis的数据类型

字符串 string

redis的一个key对应着一个string类型的value对象

哈希 hash

redis的一个key对应着field和value的映射表(javascript中的对象就是一个hash)

列表 list

列表是redis中简单的字符串列表，按照插入的顺序对列表进行排序，插入可以向前插入向后插入

集合 set

集合是redis中string类型的无序集合，且不允许有重复的成员

有序集合 zset

同set类型一样，同样是string类型的集合，且不允许有重复的成员，但是每个成员都会关联一个score分数，redis通过分数从小到大排序

redis键命令

一些常用的命令概览

set

设置键，在redis 2.6.12版本时set命令加入了其他的选项

  • EX 过期时间，单位s
  • PX 过期时间，单位ms
  • NX 键不存在时，才会设置键
  • XX 键存在时，才会设置键

del

在键存在的时候，删除键

ttl, pttl

返回键的剩余的过期时间

rename

修改键名

type

返回键值的类型

redis string命令

set, mset

设置键值

get, mget

获取键值

strlen

获取值的长度

redis hash命令

hdel

删除一个或者多个哈希表中的字段

hget, hmget

获取一个或者多个哈希表中的字段的value

hset, hmset

设置一个或者多个哈希表中的字段的value

hkeys, hvals, hlen

hkeys获取hash表中的所有字段，hvals获取hash表中所有字段对应的值，hlen获取hash表中字段的长度

redis事务

redis事务，可以一次执行redis的一组命令。事务中所有命令都会序列化，按照顺序执行。同时redis事务中的命令要么全部被执行，要么全部都不执行。

multi用于开启一个事务，multi开始事务后，客户端可以继续添加redis命令，但是这些命令不会立刻执行，而是被放到一个队列中，但使用exec命令的时候，所有队列中的命令才会被执行。DISCARD则可以清空事务队列，放弃执行redis事务。

如果事务在执行过程中产生了错误，redis并没有对它们进行特别处理, 即使事务中有某个/某些命令在执行时产生了错误，事务中的其他命令仍然会继续执行。

乐观锁🔒

watch命令可以为redis事务提供乐观锁的功能，在watch命令执行之后，watch会对指定key进行监控，如果其他的客户端修改了监控的key。那么当前客户端的事务必然会失败。watch命令让事务只能在所有被监视键都没有被修改的前提下执行，如果这个前提不能满足的话，事务就不会被执行。对键的监控从watch命令执行开始生效，exec命令执行失效（事务的无论成功与否，exec的执行都会让监控失效）
