http协议基础知识(3)

发布于 2018-04-08

响应首部字段

Accept-Ranges

告知客户端，服务器端能否处理范围请求

Age

如果创建Age字段的是源服务器，代理服务器返回，告知客户端源服务器在多久之前创建的响应。

如果创建Age字段的是代理服务器，Age字段表示的是，缓存后到再次验证缓存有效性的时间值。

ETag

告知客户端，资源的实体标示（字符串形式，资源的唯一性标示）。服务器会为每一份资源创建ETag值。生成ETag，没有统一的算法。只是由服务器为资源分配ETag值。

URI，相同的资源ETag值并不相同（比如中文的谷歌，和英文的谷歌）。资源发生更新后，URI地址不会改变，但是ETag的值会发生更新。

ETag又分为强ETag和弱ETag，强ETag无论资源发生多小的变化，ETag都会更新。弱ETag相反。

Location

会将客户端引导至某个与请求URI位置不同的资源。配合3xx, 提供资源重定向的功能。会让浏览器对Location指定的地址，再次发起请求。

Retry—After

告知客户端，在多久之后，再次发起请求。

Server

告知客户端，当前服务器HTTP应用的程序信息

Vary

源服务器对代理服务器使用Vary字段，如果代理服务器需要缓存，仅仅会对请求中包含Vary指定字段，并且指定字段的内容相同的请求进行缓存。

实体首部字段

是在请求报文，和响应报文中，实体部分使用的首部。

Allow （响应实体首部字段）

告知客户端，服务器端支持的请求方式。

Content-Encoding (响应实体首部)

告知客户端，服务器对实体的主体部分选用的编码格式，指的是对实体部分的压缩格式（gzip等）

Content——Language (响应实体首部)

告知客户端，实体主体使用的自然语言（中文，日文，英文等）

Content-Length （响应实体首部）

告知客户端，实体主体的大小（单位为字节）

Content-Location

告知客户端，报文主体对应的URI地址。（因为请求的可能是A地址，但是返回的可能是B地址的内容）

Content—MD5（响应实体首部）

客户端对接收的报文主体进行MD5处理后，转为Base64码。与响应实体首部的Content-MD5字段对应的值进行对比。验证报文主体的准确性。

响应实体的Content-MD5，就是报文主体经过MD5处理之后的内容

Content-Range

主要针对客户端的范围请求，告知客户端，响应实体的具体所在的范围。

Content-Type （响应实体首部）

告知客户端，响应实体主体的对象类型，文本类型, html类型

Expires

缓存服务器，在接收到含有Expires的响应的时候，如果资源的缓存时间在Expires指定的时间之后，缓存服务器会向源服务器请求资源。

Last—Modified

告知客户端，资源最后的修改时间

Cookie相关首部字段

Set—Cookie（cookie响应首部字段）

服务器管理客户端的Cookie的字段

  1. NAME = VALUE, Cookie的key和value
  2. expires=DATE, Cookie的有效期
  3. path=PATH，
  4. domain=域名，Cookie适用的域名（可以使用二级域名）
  5. secure属性，限制页面仅在HTTPS链接时候，才可以发送Cookie
  6. HttpOnly，会让javascript的脚本无法获取Cookie（Web页面可以读取Cookie，但是无法设置Cookie）

Cookie （cookie请求首部字段）

发送给服务器端的Cookie的内容
