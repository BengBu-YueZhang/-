1.3.3 Koa-compose

去除了一些注释和边界判断，Koa-compose的源码如下所展示的。

在学习redux中，redux中间件的本质是通过循环中间件数组，对redux提供了的dispatch方法，进行了一层又一层的包装（dispath早已经不是曾经的dispath了，是被包装了很多层的dispath）。每次通过dispath发送的action，都会经过包装的dispath依次的处理后，交给原来的dispath，然后提交给redux的reducer。其实在Koa的中间件的原理也是类似的😄（Koa中是通过递归包装dispatch，而不是循环中间间数组）

1.3.4 Koa与Context

在Koa文档中是这样描述的Context的，Koa Context 将 node 的 request 和 response 对象封装到单个对象中，为编写 Web 应用程序和 API 提供了许多有用的方法。 这些操作在 HTTP 服务器开发中频繁使用，它们被添加到此级别而不是更高级别的框架，这将强制中间件重新实现此通用功能。

1.3.5 Koa与Request请求对象

Koa Request 对象是在 node 的请求对象之上的抽象，提供了诸多对 HTTP 服务器开发有用的功能。
