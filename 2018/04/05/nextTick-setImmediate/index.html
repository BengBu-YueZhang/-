浏览器的事件循环机制 ♻️

宏任务：script,setTimeout,setInterval,IO,交互

微任务：promise

总结为一句话：执行宏任务，清空微任务队列，执行一个宏任务，清空微任务队列

⚠️浏览器事件循环需要注意的

  1. Promise中，先遇到then的先进入任务队列
  2. 如果微任务一直被添加，则会继续执行微任务，“卡死”宏任务。

Nodejs中事件循环 ♻️

Nodejs中，只有当主线程执行完成后，才会轮询任务队列中的任务。

Nodejs中Event Loop的6个阶段

  1. timer阶段：执行到期的setTimeout，setInterval
  2. IO：检查是否有已完成的IO操作的回调，以及来自上一轮poll阶段的残留（上一次可能因为时间过长或者timer时间到了提前结束的poll阶段，所残留的）
  3. idle，prepare：可忽略内部机制
  4. poll阶段：等待还没完成的I/O事件，并执行IO事件队列。ps：会因timers和超时时间等结束等待，如果timer阶段异步操作完成，会结束等待，依次经过check，close阶段重新进入timer阶段
  5. check：执行setImmediate的回调
  6. close：执行一些onClose事件

关于process.nextTick()

process.nextTick(), 会在不同的每次队列清空后执行

process.nextTick并不属于 event loop的一部分，不管event loop处在哪一个部分，都会在当前阶段之后执行。（清空当前阶段的队列后，就会立刻执行）

Node中event loop执行的过程

执行主线程
  1. 同步任务
  2. 发出异步请求
  3. 规划定时器生效的时间
  4. 执行process.nextTick()
进入事件循环
  1. 如果setTimeout，setInInterval完成，回调用进入timer队列 | 清空timer队列；清空process.nextTick()；清空微任务队列
  2. 如果IO操作完成，清空IO队列；清空process.nextTick()；清空微任务队列
  3. 清空 setImmediate；清空process.nextTick()；清空微任务队列
  4. 清空 Close队列；清空process.nextTick()；清空微任务队列
  5. 进入下一轮循环

优先级：

process.nextTick > setTimeout, setInterval > setImmediate

各个阶段的细节

timers阶段

会执行setTimout，setInterval到期，进入timers队列的任务

I/O

会执行上一次残留的IO队列，以及系统操作的回调

poll

poll会等待IO操作完成，执行IO的队列。如果发现计时器的时间到了，就绕回到 timers 阶段执行计时器的回调。

event loop经过poll阶段可能有以下两种情况：

  1. 如果 poll 队列不是空的，event loop 就会依次执行队列里的回调函数，直到队列被清空或者到达 poll 阶段的时间上限
  2. 如果 poll 队列是空的，就会：
    • 如果有 setImmediate() 任务，event loop 就结束 poll 阶段去往 check 阶段。
    • 如果没有 setImmediate() 任务，event loop 就会等待新的回调函数进入 poll 队列，并立即执行它。
  3. 一旦 poll 队列为空，event loop 就会检查计时器有没有到期，如果有计时器到期了，event loop 就会回到 timers 阶段执行计时器的回调。

check

这个阶段允许开发者在 poll 阶段结束后立即执行一些函数。如果 poll 阶段空闲了，同时存在 setImmediate() 任务，event loop 就会进入 check 阶段。

如果一个回调是由 setImmediate() 发出的，同时 poll 阶段空闲下来了，event loop就会结束 poll 阶段进入 check 阶段，不再等待新的 poll 事件。

setTimeout 和 setImmediate

⚠️⚠️👮：如果存在setImmediate，会在poll阶段结束后立即执行

process.nextTick()

会在任何阶段的队列清空后立即执行。进入下一个event loop阶段前
